Anthropic released [Claude Opus 4.7](https://www.anthropic.com/news/claude-opus-4-7) on April 16. Compared to Opus 4.6, it improves autonomous software engineering — longer-running coding tasks, better self-verification of outputs, and stricter instruction-following. Vision input resolution increases to ~3.75 megapixels (3× previous limit), and document reasoning errors drop 21% on enterprise benchmarks. A new `xhigh` effort level sits between `high` and `max`. Pricing stays at $5/$25 per million tokens.

Behavioral changes worth knowing: response length now scales with task complexity instead of defaulting long, the model reasons more while calling tools less often, and the updated tokenizer increases token counts by roughly 1.0–1.35×. Stricter instruction-following means prompts tuned for Opus 4.6 may need adjustment. Anthropic published a [best practices guide for Opus 4.7 with Claude Code](https://claude.com/blog/best-practices-for-using-claude-opus-4-7-with-claude-code) (short version: provide full context upfront, delegate like you would to an engineer) and a [migration guide](https://platform.claude.com/docs/en/about-claude/models/migration-guide#migrating-to-claude-opus-4-7) covering API and tokenizer changes.

Key points:

- **Coding benchmarks** — double-digit improvements over Opus 4.6; better at long-running tasks without human supervision
- **Vision** — 3× higher input resolution; visual acuity benchmark goes from 54.5% (4.6) to 98.5% (4.7)
- **Effort levels** — new `xhigh` setting recommended for most coding and agentic work
- **Prompt compatibility** — stricter instruction-following may require re-tuning existing prompts
- **Same pricing** — $5 input / $25 output per million tokens; available on API, Bedrock, Vertex, and Foundry
