Alibaba Cloud's Qwen Team has open-sourced **Qwen3.6-35B-A3B**, a sparse mixture-of-experts (MoE) model with 35 billion total parameters and only **3 billion active parameters**. Despite activating less than 10% of its weights per forward pass, it delivers agentic coding performance that rivals much larger dense models.

## Why It Matters

The model's headline achievement is efficiency without sacrifice. With 3B active parameters it beats Qwen3.5-27B (a dense model nearly 10× larger by active compute) on several coding benchmarks and dramatically outpaces its direct predecessor Qwen3.5-35B-A3B:

| Benchmark | Qwen3.5-35B-A3B | Qwen3.5-27B | **Qwen3.6-35B-A3B** |
|---|---|---|---|
| SWE-bench Verified | 70.0 | 75.0 | **73.4** |
| Terminal-Bench 2.0 | 40.5 | 41.6 | **51.5** |
| QwenWebBench (Elo) | 978 | 1068 | **1397** |
| AIME 2026 | 91.0 | 92.6 | **92.7** |
| GPQA | 84.2 | 85.5 | **86.0** |

Terminal-Bench 2.0 (+11 points over predecessor) and QwenWebBench (+40% Elo gain) show the sharpest jumps — exactly the agentic and frontend coding tasks where real-world usage lives.

## Natively Multimodal

Qwen3.6 is a vision-language model by default, not a text-only model with a bolted-on vision encoder. At 3B active parameters it matches or beats Claude Sonnet 4.5 on most VL benchmarks:

| Benchmark | Claude Sonnet 4.5 | Qwen3.6-35B-A3B |
|---|---|---|
| RealWorldQA | 70.3 | **85.3** |
| MMBenchEN-DEV v1.1 | 88.3 | **92.8** |
| VideoMMMU | 77.6 | **83.7** |
| RefCOCO (avg) | — | **92.0** |

Spatial intelligence (RefCOCO, ODInW13) is a particular strength, which matters for agents that need to interpret screenshots and UI layouts.

## Thinking Mode and Agentic Use

Like its predecessor, Qwen3.6-35B-A3B supports both **thinking** (chain-of-thought) and **non-thinking** modes in a single checkpoint. The API exposes `enable_thinking` and a `preserve_thinking` flag — the latter carries reasoning traces across multi-turn agentic conversations, which the team recommends for tool-use and long-horizon tasks.

## Running It with Claude Code

Qwen's API exposes an **Anthropic-compatible endpoint**, meaning you can point Claude Code directly at Qwen3.6-Flash with four environment variables:

```bash
export ANTHROPIC_MODEL="qwen3.6-flash"
export ANTHROPIC_SMALL_FAST_MODEL="qwen3.6-flash"
export ANTHROPIC_BASE_URL=https://dashscope-intl.aliyuncs.com/apps/anthropic
export ANTHROPIC_AUTH_TOKEN=<your_dashscope_api_key>
claude
```

This gives you the full Claude Code agentic experience — file editing, bash tools, multi-turn reasoning — powered by a model that costs a fraction of frontier pricing and can also be self-hosted.

## Availability

- **Qwen Studio** — chat at qwen.ai
- **API** — as `qwen3.6-flash` on Alibaba Cloud Model Studio (OpenAI-compatible and Anthropic-compatible endpoints)
- **Open weights** — Hugging Face and ModelScope (self-host with Transformers, vLLM, llama.cpp, LM Studio, etc.)

The weights are fully open, not gated. A model with 3B active parameters running competitively with dense 27B+ models on agentic benchmarks is a meaningful milestone for the open-source ecosystem.
